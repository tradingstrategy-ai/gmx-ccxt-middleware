# Configuration

The GMX CCXT Middleware Server is configured entirely through environment variables.

The authoritative implementation lives in [config.py](../src/gmx_ccxt_server/config.py), where `load_config_from_env()` reads the process environment and maps it into the runtime settings passed to the Python GMX exchange. The checked-in container defaults live in [docker-compose.yaml](../docker-compose.yaml).

## Environment variables

| Variable | Default | Short description | Long description |
| --- | --- | --- | --- |
| `GMX_SERVER_ADDRESS` | `127.0.0.1:8000` in local runtime, `0.0.0.0:8000` in the Docker image | Server bind address | Full `host:port` bind address for the FastAPI GMX CCXT Middleware Server. The parsing and validation logic is implemented in [`ServerSettings.normalize_address()`](../src/gmx_ccxt_server/config.py). The Docker image defaults this to `0.0.0.0:8000` so the container can listen on its internal interface; see [Dockerfile](../Dockerfile). |
| `GMX_SERVER_AUTH_TOKEN` | unset | Bearer token for server endpoints | Optional bearer token checked on the public server routes. When set, clients must send `Authorization: Bearer ...` to call the HTTP endpoints. The token is loaded in [config.py](../src/gmx_ccxt_server/config.py) and enforced through the route authentication code in [`auth.py`](../src/gmx_ccxt_server/routes/auth.py). |
| `GMX_LOG_LEVEL` | `info` | Python logging verbosity | Controls GMX CCXT Middleware Server process logging verbosity, passed through to Uvicorn startup in [`__main__.py`](../src/gmx_ccxt_server/__main__.py). Useful values are standard Python logging levels such as `debug`, `info`, `warning`, and `error`. |
| `GMX_RPC_URL` | `https://arb1.arbitrum.io/rpc` | Arbitrum JSON-RPC endpoint | Primary RPC endpoint used by the Python GMX exchange for chain reads and transaction submission. This value is loaded in [config.py](../src/gmx_ccxt_server/config.py) and forwarded to the `eth_defi` GMX exchange as `rpcUrl`. For architecture context, see [architecture.md](architecture.md). |
| `GMX_PRIVATE_KEY` | unset | Signing key for live orders | Optional private key used to sign GMX transactions. When present, the GMX CCXT Middleware Server creates a signing-capable Python GMX exchange and can place or cancel orders. The environment variable is mapped in [config.py](../src/gmx_ccxt_server/config.py), then used by the GMX CCXT exchange in [`exchange.py`](../web3-ethereum-defi/eth_defi/gmx/ccxt/exchange.py). Treat this as sensitive secret material. |
| `GMX_WALLET_ADDRESS` | unset | Read-only wallet address | Optional wallet address for account-scoped reads when no private key is configured. In read-only mode this lets the GMX CCXT Middleware Server query balances, positions, and account-related state for a target address. The runtime avoids overriding the signing wallet address when `GMX_PRIVATE_KEY` is present; see [runtime.py](../src/gmx_ccxt_server/runtime.py). |
| `GMX_CHAIN_ID` | unset | Explicit chain override | Optional explicit chain ID passed through to the Python GMX exchange. Normally the exchange can infer the chain from the RPC endpoint, but this setting allows an explicit override when needed. Loaded in [config.py](../src/gmx_ccxt_server/config.py) and forwarded as `chainId`. |
| `GMX_SUBSQUID_ENDPOINT` | unset | Custom Subsquid GraphQL endpoint | Optional override for the GMX Subsquid endpoint used for indexed order and position lookups. If unset, the default `eth_defi` chain mapping is used; if set, it is forwarded as `subsquidEndpoint`. For the official GMX indexer docs, see the [GMX Subsquid docs](https://docs.gmx.io/docs/api/subsquid/). |
| `GMX_EXECUTION_BUFFER` | `2.2` | Execution fee safety multiplier | Multiplier applied to the base execution fee when building GMX orders. In practice this increases the fee budget so keepers are more likely to execute the order even if gas conditions worsen. The GMX CCXT Middleware Server loads the value in [config.py](../src/gmx_ccxt_server/config.py); the underlying fee logic and warnings live in [`gas_utils.py`](../web3-ethereum-defi/eth_defi/gmx/gas_utils.py). For behavioural context, see the `Execution buffer` section in [architecture.md](architecture.md). |
| `GMX_DEFAULT_SLIPPAGE` | `0.003` | Default order slippage tolerance | Default slippage percentage forwarded to the Python GMX order builder. This becomes the per-order default unless the caller overrides it in the order params. The mapping is defined in [config.py](../src/gmx_ccxt_server/config.py) and consumed by the `eth_defi` GMX exchange in [`exchange.py`](../web3-ethereum-defi/eth_defi/gmx/ccxt/exchange.py). |
| `GMX_VERBOSE` | `false` | Enable verbose GMX logging | Turns on lower-level GMX Python logging for debugging. This is forwarded into the GMX exchange parameters and used by the exchange initialisation code in [`exchange.py`](../web3-ethereum-defi/eth_defi/gmx/ccxt/exchange.py). It is helpful during debugging, but generally too noisy for routine production use. |
| `GMX_PRELOAD_MARKETS` | `false` | Load markets during startup | When enabled, the GMX CCXT Middleware Server loads exchange markets during startup instead of waiting for the first client request. This can make the first market-data call faster at the cost of slightly slower process start-up. The behaviour is implemented in [runtime.py](../src/gmx_ccxt_server/runtime.py). |
| `GMX_REST_API_MODE` | `true` | Enable REST-assisted market-data mode | Enables the GMX REST-assisted market-data path in the Python GMX exchange. The GMX CCXT Middleware Server forwards this as part of the exchange `options` map in [config.py](../src/gmx_ccxt_server/config.py). The REST integrations it affects are described in the [GMX REST API docs](https://docs.gmx.io/docs/api/rest/) and summarised in [architecture.md](architecture.md). |
| `GMX_GRAPHQL_ONLY` | `false` | Force GraphQL-only market-data mode | Forces the Python GMX exchange to prefer the GraphQL/indexed data path for market loading and related reads. This is forwarded as an exchange option from [config.py](../src/gmx_ccxt_server/config.py). It is mainly useful for experiments or troubleshooting around data-loading behaviour. |
| `GMX_DISABLE_MARKET_CACHE` | `false` | Disable the GMX market cache | Disables the Python GMX market cache so market metadata is refetched more aggressively. This is also forwarded as an exchange option from [config.py](../src/gmx_ccxt_server/config.py). Use it when debugging stale metadata problems, but note that it may increase startup latency and repeated remote calls. |
| `GMX_VAULT_ADDRESS` | unset | Optional Lagoon vault address | Optional vault address forwarded into the Python GMX options so the exchange can operate in Lagoon-style vault mode. This is mapped in [config.py](../src/gmx_ccxt_server/config.py) and ties into the optional vault wallet support described in [`wallet.py`](../web3-ethereum-defi/eth_defi/gmx/lagoon/wallet.py). |

## Notes

- Empty strings are normalised to “unset” for the optional string fields in [config.py](../src/gmx_ccxt_server/config.py).
- Boolean-like and numeric values are parsed by Pydantic when the environment is loaded through `load_config_from_env()`.
- The easiest way to see all container defaults in one place is still [docker-compose.yaml](../docker-compose.yaml).
